## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ yarn install
```

## Compile and run the project

```bash
# development
$ yarn run start

# watch mode
$ yarn run start:dev

# production mode
$ yarn run start:prod
```

## Run tests

```bash
# unit tests
$ yarn run test

# e2e tests
$ yarn run test:e2e

# test coverage
$ yarn run test:cov
```


# Backend Checkpoint API

Una API desarrollada con **NestJS**, **Prisma** y **SQLite** para gestionar usuarios, camiones, ubicaciones y órdenes. Este proyecto permite a los usuarios crear órdenes vinculadas a camiones, ubicaciones (origen y destino), y gestionar su estatus.

---

## Descripción

Esta API tiene como objetivo resolver las siguientes necesidades:
- **Usuarios**: Registro, autenticación y gestión.
- **Camiones (Trucks)**: Gestión de camiones vinculados a usuarios.
- **Ubicaciones (Locations)**: Origen y destino de las órdenes.
- **Órdenes (Orders)**: Gestión de órdenes, incluyendo relaciones con usuarios, camiones y ubicaciones.

**Nota importante:**  
Inicialmente, se planeó usar la API de Google Maps para obtener coordenadas y direcciones basadas en `place_id`. Sin embargo, debido a restricciones de facturación, esta funcionalidad fue implementada manualmente y no utiliza la API de Google Maps.

---

## Tecnologías Utilizadas

- **NestJS**: Framework modular para aplicaciones de Node.js.
- **Prisma**: ORM para la gestión de la base de datos.
- **SQLite**: Base de datos ligera y fácil de usar.
- **Class Validator**: Validación de datos en DTOs.
- **TypeScript**: Tipado estático para un desarrollo más robusto.
